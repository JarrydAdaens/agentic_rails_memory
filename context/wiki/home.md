## Context as a Working Language

In an agentic IDE, context is one of the programming languages of the workflow. Markdown,
YAML, XML documentation comments, and Doxygen comments are not just file formats. They are
ways of expressing structure, intent, constraints, examples, and meaning so both humans and
agents can reason over them clearly.

Wielding context thoughtfully has a direct effect on the quality of outcomes. Clear
headings, stable structure, explicit examples, and deliberate wording make the difference
between vague guidance and reliable execution. This matters doubly for a memory store:
every artifact deposited here is future context for some other agent.
